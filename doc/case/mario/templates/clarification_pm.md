# 澄清模板：产品经理（范围与验收）

用途：用于“澄清问题收集”命令的输出模板。重点是定义范围边界、验收标准、交付形式与风险约束，确保 DAG 可生成且门禁可判定。

## 输出要求（结构化）

每个问题请包含：
- 问题
- 为什么必须问（对应的门禁/验收/范围锁定）
- 关联交付物/任务
- 默认假设（若用户不答）
- 风险
- 优先级：P0/P1/P2

## 建议问题清单（示例）

1. 目标与成功标准
   - “完成”的定义是什么？可玩 demo / 可发布 / 可扩展框架？
   - 主要评估指标：可玩性、稳定性、性能、关卡数量？
2. 交付物与运行方式
   - 交付形式：可执行文件/网页链接/源码仓库/zip？
   - 是否需要安装包、版本号、发布说明？
3. 范围与时间
   - 时间预算/迭代周期？是否允许先交 MVP 再迭代？
   - 明确不做什么（例如：多人、存档、多语言、复杂剧情）？
4. 验收与门禁
   - 必须通过哪些门禁：构建、smoke、e2e、性能、兼容性？
   - 验收证据需要什么格式（报告/截图/录屏/日志）？
5. 风险与合规
   - 是否明确不使用第三方受版权保护素材？素材来源策略？
   - 是否需要安全扫描、依赖许可证清单？

## 交付格式建议

- 用 `clarification_questions_pm.md`（或 `.json`）输出。
- 输出应能直接映射到 `plan_manifest.json` 的 `deliverables` 与 `policies.release_gates_required`。
