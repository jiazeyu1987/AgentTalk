# Mario 需求/验收标准评分模板 v1

criteria_version: `mario_requirements_rubric.v1`

用途：用于需求制定与验收标准评审（`score_required=true`），让“需求质量”可量化。

## 总分与判定

- 总分：0-100
- 建议阈值：
  - `PASS/APPROVE`: >= 80
  - `REVISE`: 50-79
  - `REJECT`: < 50

## 分项（每项 0-20，共 5 项）

1) MVP 范围清晰（0-20）
- 20：明确 MVP 必做/不做、关卡数量、敌人/机关范围、UI/音效范围
- 10：范围大致清楚但边界仍模糊
- 0：范围不可落地或频繁自相矛盾

2) 操控与手感指标（0-20）
- 20：平台与输入方式明确；关键手感参数有描述（跳跃、加速度、容错）
- 10：有描述但缺少可验收的指标
- 0：完全没有手感/操控定义

3) 关卡与内容约束（0-20）
- 20：关卡目标、长度、机关、敌人类型清晰；可生成关卡数据
- 10：关卡方向明确但细节不足
- 0：无法指导关卡制作

4) 验收标准可执行（0-20）
- 20：每个关键功能都有验收条款（可用录屏/日志/测试报告证明）
- 10：验收条款存在但不可操作
- 0：没有验收标准或全是主观描述

5) 素材与合规（0-20）
- 20：素材来源策略明确（占位/开源/自制），版权/商标风险有约束
- 10：有原则但缺执行细则
- 0：存在明显侵权风险或完全未声明

## 输出格式要求

必须输出：
- `criteria_version`
- `score`
- `score_breakdown[]`（五项）
- `issues[]`（缺口列表）+ `suggestions[]`（可执行补齐建议）
