# Q020: 缺少错误恢复机制

**问题ID**: Q020
**类别**: 可靠性
**严重程度**: 🟡 中等

---

## 问题描述

Agent执行任务失败后，没有机制定义如何恢复和重试。

## 场景示例

Backend Dev执行任务时失败：
- 原因：API密钥过期
- 错误记录到error_chains.jsonl

问题：
1. ❓ GM如何知道Backend Dev失败了？
2. ❓ GM是否应该自动重试？重试几次？
3. ❓ 是否可以分配给其他Agent？
4. ❓ 如果无法恢复，如何通知用户？
5. ❓ 如何记录失败原因，便于后续分析？

## 影响

- 任务失败后无法恢复
- 整个DAG可能卡住
- 用户体验差

## 关键问题

1. 如何检测Agent任务失败？
2. 自动重试策略是什么？
3. 是否可以重新分配给其他Agent？
4. 如何通知用户处理？
5. 如何记录和学习失败原因？

## 解决方向

需要定义错误恢复策略
