# Q018: 缺少任务进度跟踪机制

**问题ID**: Q018
**类别**: 任务协调
**严重程度**: 🟡 中等

---

## 问题描述

GM分配多个子任务后，没有机制跟踪每个任务的进度。

## 场景示例

GM分配了5个子任务，分配给3个Agent：

用户问："现在整体进度如何？"
1. ❓ GM如何知道每个子任务的状态（待执行/执行中/已完成/失败）？
2. ❓ GM如何计算整体进度百分比？
3. ❓ GM如何向用户报告进度？
4. ❓ 如何检测任务卡住（长时间无更新）？

## 影响

- 无法向用户报告进度
- 无法及时发现问题
- 用户体验差

## 关键问题

1. 如何收集每个Agent的任务状态？
2. 如何定义任务的执行阶段？
3. 如何计算整体进度？
4. 如何检测任务卡住？
5. 如何向用户展示进度？

## 解决方向

需要定义PR-022: 任务协调机制的一部分
