# Q012: 缺少LLM调用机制

**问题ID**: Q012
**类别**: 执行机制
**严重程度**: 🔴 致命

---

## 问题描述

PR-007定义了统一的执行命令，提到"提交给LLM获取返回值"，但没有定义如何调用LLM。

## 场景示例

GM收到复杂任务需要分解：
1. GM给自己发送执行命令：分解任务
2. 命令的prompt要求："将任务分解为多个子任务"
3. ❌ GM不知道如何将prompt提交给LLM
4. ❌ GM不知道如何获取LLM的返回值

## 影响

- Agent无法利用LLM的能力
- 无法进行任务分解、决策等需要LLM的操作
- 核心的"LLM驱动"理念无法实现

## 关键问题

1. Agent如何连接LLM服务（API密钥、端点）？
2. 如何构造LLM请求（prompt + inputs）？
3. 如何处理LLM响应（解析返回值、提取分数）？
4. LLM调用失败如何重试？
5. 如何记录LLM调用日志（tokens、费用）？

## 解决方向

需要定义PR-019: LLM调用机制
