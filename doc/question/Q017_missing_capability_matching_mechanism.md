# Q017: 缺少能力匹配机制

**问题ID**: Q017
**类别**: 任务分配
**严重程度**: 🟡 中等

---

## 问题描述

PR-009提到"将任务划分给必要的Agent"，但没有定义如何判断Agent能力与任务需求是否匹配。

## 场景示例

GM需要分配任务"设计数据库schema"：

visibility_list中有：
- PM Agent: 需求分析、产品设计
- DB Expert: 数据库设计、性能优化
- Backend Dev: 后端开发、API设计

问题：
1. ❓ GM如何知道DB Expert最适合这个任务？
2. ❓ 如果DB Expert不可用（忙碌/离线），是否可以分配给Backend Dev？
3. ❓ 如何量化匹配度？
4. ❓ 如果多个Agent都能做，如何选择？

## 影响

- 可能分配给错误的Agent
- 任务执行效率低
- 可能导致任务失败

## 关键问题

1. 如何比较任务需求与Agent能力？
2. 如何计算匹配度分数？
3. 如何处理多个候选Agent的情况？
4. 如何处理Agent不可用的情况？
5. 是否需要人工确认分配结果？

## 解决方向

需要定义能力匹配算法
