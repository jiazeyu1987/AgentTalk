# Q015: 缺少DAG执行机制

**问题ID**: Q015
**类别**: 任务协调
**严重程度**: 🟡 严重

---

## 问题描述

PR-009提到GM输出DAG图描述任务依赖关系，但没有定义如何按DAG执行任务。

## 场景示例

GM生成DAG：
- task_001 (需求分析) → PM Agent
- task_002 (数据库设计) → DB Expert，依赖task_001
- task_003 (后端开发) → Backend Dev，依赖task_001和task_002

问题：
1. ❓ GM如何知道task_001已完成？
2. ❓ GM如何等待task_001完成后才开始task_002？
3. ❓ 如果task_001失败，task_002和task_003如何处理？
4. ❓ GM如何跟踪整体进度？

## 影响

- 无法实现任务依赖
- 可能导致任务执行顺序错误
- 无法追踪整体进度

## 关键问题

1. 如何检测一个任务是否完成？
2. 如何等待前置任务完成？
3. 如何处理DAG中的并行任务？
4. 前置任务失败时，后续任务如何处理？
5. 如何向用户报告整体进度？

## 解决方向

需要定义PR-022: 任务协调机制的一部分
