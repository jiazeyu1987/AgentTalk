# Q022: 缺少任务优先级机制

**问题ID**: Q022
**类别**: 调度
**严重程度**: 🟢 低

---

## 问题描述

当GM同时收到多个任务时，没有机制定义执行顺序和优先级。

## 场景示例

GM同时收到3个任务：
1. 任务A：紧急bug修复（高优先级）
2. 任务B：新功能开发（中优先级）
3. 任务C：文档更新（低优先级）

问题：
1. ❓ GM如何知道任务的优先级？
2. ❓ 如何中断低优先级任务，处理高优先级任务？
3. ❓ 任务优先级如何定义？
4. ❓ 是否支持动态调整优先级？

## 影响

- 紧急任务可能被延迟
- 资源分配不合理
- 用户体验差

## 关键问题

1. 任务优先级如何定义和传递？
2. GM如何根据优先级调度任务？
3. 是否支持抢占（中断低优先级任务）？
4. 优先级如何动态调整？
5. 同优先级任务的调度策略是什么（FIFO？）？

## 解决方向

需要定义任务优先级和调度机制
